一句话定位

把散落在 Excel / Word 里的客户地图，变成可跟踪的「单位 → 人 → 公示项目 → 商机」一条线； 并对每条商机做 A/B/C 自动分级 和决策链、竞争、行动、成交条件、跟进的全流程管理。

核心功能模块

① 客户地图导入
Excel(.xlsx/.xls) + Word(.docx) 双源导入，按单位名去重合并；支持「整文件=1 单位」「按 sheet=单位」「按行=单位」三种粒度。
② 客户 360° 视图
单位信息 + 联系人 + 公示项目 + 关联商机 + 跟进时间线，一页看全。
③ 商机 A/B/C 分级
我方产品(+2) + 招标时间(+2) + 预算(+1) + 联系人(+1)，自动定级，超 7 天未跟进标红。
④ 公示项目 → 转商机
监控到的采购公示一键转成商机并双向关联，对接人跟着挂上。
⑤ 决策链 MEDDPICC
记录签字人 / 预算 Owner / 技术评审 / Champion 等角色、态度、影响力。
⑥ 竞争 / 行动 / 成交条件
竞争态势评分、行动推进清单、成交条件核对、跟进记录。
⑦ 产品库
导入 Revvity 设备清单（进口线 + 国产新波线），新建商机时按产品线联动选具体型号、自动带金额。
⑧ 新建/编辑联动
所属单位、对接人下拉选择 + 可新建，自动回写客户关联，告别纯文本乱填。

AI 在开发里到底干了啥（关键 Bug 修复记录）

这套系统不是「一次生成就完事」，而是在多轮对话里边写边测、逐个排雷。以下是 AI 定位并修复的几个典型问题：

docx 客户地图漏导入：原代码用 '负责销售' in hdr（hdr 是列表）做判断，实际单元格是「负责销售（没有可空着）」，精确匹配失败 → 整批中医药单位被静默跳过；另农口 docx 表头在第 1 行而非第 0 行。改为字符串包含判断 + 动态定位表头后，单位数从 23 恢复到 150。
INSERT 列值错位（数据污染）：save_project 的 INSERT 列顺序与值元组顺序不一致，从第 13 列起全部错位 —— customer_id/contact_id 写成时间戳、单位/联系人 id 错写进备注字段。修复后新建商机的客户关联全部正确，并修正了被污染的存量记录。
Excel 导入列映射错：原导入把「序号」列当客户名，产生 991 条垃圾；真实线索仅 12 条。改为读正确的客户名称列并按真实列映射。
其它：导入未建表保护、同名线索被去重吞掉、等级写死不自动算、表单裸 st. 前缀报错等，均已修复。

结论：AI 的价值不止在「写初版」，更在「能跑通 + 能排错 + 能端到端验证」。每个修复都有冒烟测试回读落库值确认。

怎么跑起来

环境用本机托管的 Python（已装 streamlit / pandas / openpyxl）：

python -m streamlit run crm_tracker.py --server.port 8501

浏览器打开 http://localhost:8501。左侧导航：仪表盘 / 新建商机 / 客户管理 / Excel导入 / 使用说明。

数据全部在本地单文件 crm_data.db（SQLite），不上云、不依赖任何外部服务。

数据说明（为什么这页没有客户名单）

本页是「项目情况 / 开发记录」的脱敏概览。系统里实际沉淀的是真实客户地图（单位、联系人姓名、研究方向等），属于商业敏感信息， 不随源码或本页上传到任何公开仓库。如需对外展示，只展示聚合量（如上面的数字）与功能架构，不暴露可识别个人 / 单位明细。
